Developer Notes (what to expect) 
- Single file: all logic lives here; worker is inlined via Blob.
- Configurable relay: set ?relay=ws://host:port or type into the Relay field.
- Session: ?session=abc123 (auto-generated if missing).
- Worker responsibilities:
  • Queue outbound events until WS is open; bounded buffer (50).
  • Heartbeat ping/pong; reconnect with backoff.
  • Mirrors to BroadcastChannel for same-device tabs.
  • Enforces message shape: {session,type,ts,value} and drops stale/unknown.
- Main thread only handles UI + audio; all transport state is inside the worker.
- Status pill shows: connecting, buffering N, synced, retrying, or errors.
- Manual test: open two tabs with same ?session and relay, change text/color, tap Beep.
